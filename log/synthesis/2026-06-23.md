## 코드 리뷰

**1. MAE skip 누수 수정 완전성 미검증**
- `pretrain/models.py`, `pretrain/train.py` — SCRATCHPAD에서 `use_skip=False` 적용으로 수정 완료라고 하나 diff patch 30000B 상한으로 .py 코드가 잘려 실제 구현 확인 불가. ResEnc U-Net의 encoder-level skip과 ViT conv-stem skip 두 경로가 모두 차단됐는지, `test_models.py`(+46줄)에 `use_skip=False` 경로 커버가 포함됐는지 diff에서 불명확. 커밋 10c8c40 메시지로 수정 사실 자체는 지지됨.

**2. InfoNCE in-batch false negative 리스크**
- `pretrain/train.py` (dbffe11) — 3D 의료 영상에서 같은 subject의 두 crop이 동일 배치에 들어가면 false negative(positive pair를 negative로 처리)가 발생해 L\_g가 인위적으로 낮아진다. SCRATCHPAD는 "negative로 붕괴 차단"만 언급할 뿐 배치 내 subject 중복 방지 전략 기재 없음. Phase A subset 4000에서 발생 가능성 존재.

**3. emb\_std\_mean STOP 임계 0.01 — InfoNCE 모드에서 미재검증**
- `pretrain/monitor.py` (+21줄) — SCRATCHPAD에서 "code-auditor PASS(비정규화 입력이라 건강 recipe 3~36× 마진, 오발 없음)"는 DINO 모드 기준. InfoNCE는 embedding 분포가 달라(negative 밀어내기 → std 더 클 수 있음) 임계가 너무 보수적일 수 있음. InfoNCE arm 실측 마진 확인 기록 diff에 없음.

**4. DINO head revert 후 InfoNCE 전환 시 head 구조 불명확**
- `pretrain/models.py` — SCRATCHPAD: "head fix는 ckpt 호환 깨져 일단 revert". InfoNCE 모드에서 `_MLPHead`(plain MLP, L2-norm 없음)가 글로벌 경로에 그대로 연결되는지, InfoNCE용 projection head가 별도 분기인지 diff 코드가 잘려 확인 불가. plain MLP가 그대로라면 low-magnitude logit 문제가 InfoNCE 온도 항에도 영향.

**5. W3 full-finetune — "이득 0" 결론 표현 과장 (부분 지지)**
- `pretrain/seg_finetune.py`, commit 6a78a46 — Δ(pre−scratch): vit\_mae −0.025 / vit8 +0.011 CI\[.000,.188\] / resenc −0.051. 음수 방향이나 전 arm CI에 0 포함 → 검출 불가 상태. 단 커밋 메시지 자체는 "이득 미입증/음수"로 기재돼 있어 SCRATCHPAD의 "단정 못 함" 자기인식과 일치함. 초안이 "과도한 결론 표현"이라고 지적한 것은 커밋 제목의 "(이득 미입증/음수)" 단서를 생략한 채 평가한 것으로, 부분적으로만 유효.

**6. train\_health.py 호출 체인 미배선**
- `pretrain/train_health.py` (+74줄, commit fbb0a40) — 커밋 설명은 "에이전트 모니터링용; loss 반분평균으로 false-flag 방지"이나, train.py / supervisor.py에서의 import·invoke 경로가 diff에 나타나지 않음. 독립 실행 스크립트이면 자동 모니터링 chain이 불완전.

**7. polymicro log의 confound 경고는 존재하나 runtime 자동 제외 로직 미확인**
- `experiments/phase_a/baseline_v2/log.txt:17` — log에는 "⚠️ random baseline 미지정 → 절대값엔 위치/site confound 포함" 경고가 실제로 있음(초안 제목 "경고 없음"은 오기). 그러나 SUMMARY.md 판정 "(M4) polymicro cls random ViT도 0.97 → 제외"는 수동 판단으로 처리됐고, eval\_harness.py가 confound-dominated task를 런타임에 자동 경고·제외하는 로직 존재 여부는 diff에서 불명확.

**8. data.py 3-리뷰 4개 버그 수정 — 초안 미검토** *(초안 누락 항목)*
- `pretrain/data.py` (+150줄, commit 4644f8d) — SCRATCHPAD에 버그 4개 명기: ① 작은 축 zero-pad가 znorm/MAE 타깃 오염(→ min\_size 필터), ② build\_filelist 226K gpfs stat 행(→ subset만 stat), ③ stale filelist cache 헤더 fail-fast 미적용(사일런트 오염), ④ DataLoader worker leak(→ try/finally join). 각 수정의 경계조건 및 정상·crash 양방향 실측은 SCRATCHPAD에 기재됐으나 diff patch가 잘려 코드 레벨 확인 불가. 특히 ③ 헤더 fail-fast가 manifest 버전 변경 시 실제로 검출되는지 단위 테스트 커버리지 불명.

## 놓친 것·결정 필요

**1. Phase A 미완료 상태에서 Phase B 전체 가동 (가장 중요)**
SCRATCHPAD: `experiments/phase_b/resenc_infonce_full`(226K), `vit_ibot_full` 가동 중. docs/04\_strategy\_plan.md §4는 "Phase B = 검증된 recipe만 full-corpus"로 명시하나, Phase A 판정 기준인 "7-task Pareto 지배 recipe 1개 확정"은 미달이다(cls 전 무정보, trigeminal 0, resenc broken). "탐색적 조기 시작"인지 "문서 기준 변경"인지 의사결정 기록 없음.

**2. ResEnc+InfoNCE arm이 Phase A OAT에 없음**
4-arm 설계(vit\_mae/vit\_ibot/vit8\_mae/resenc\_mae)는 InfoNCE 발견 이전 확정됐으므로 ViT 대비 동일 예산 비교 불가. resenc\_mae(broken)와 resenc+InfoNCE는 다른 arm — 현재 bake-off 결과에서 CNN vs ViT 비교는 성립하지 않음. Phase A OAT 재설계 또는 InfoNCE arm 추가 실행 여부 결정 필요.

**3. vit\_ibot\_full에 skip-free 조건 적용 여부** [근거부족]
초안은 "vit\_ibot이 MAE dense loss를 포함하면 skip-free 필요"라고 했으나, diff 어디에도 vit\_ibot recipe가 MAE 재건 손실을 포함한다는 근거가 없음. SCRATCHPAD는 vit\_ibot을 "3DINO식 양축 baseline(iBOT/DINO head)"로, vit\_mae를 별도 MAE arm으로 구분 기재함. MAE skip-free 수정은 마스크 영역 재건 경로가 있는 recipe에만 적용되며 iBOT 패치 토큰 자기-증류에는 해당하지 않음. 이 concern은 비지지 전제에 기반하므로 삭제.

**4. trigeminal(task4) 해결 경로 미제시**
전 방법(patch16·patch8·crop128·frozen probe·full-finetune)에서 0. SUMMARY.md에 "고해상/shallow-nonlinear seg probe → trigeminal 측정가능성" 권고가 있으나 구체 구현 시점 없음. task4가 리더보드 7 task 중 하나이고 내부 eval 불가면 recipe 선택 근거 부재.

**5. 커밋 해시 중복** ~~[삭제]~~ → **오기 정정**
초안이 동일 해시로 지목한 두 커밋은 실제로 `7c46d1b`(FIX monitor)와 `7c6a719`(EVAL seg-finetune 하네스)로 서로 다른 해시임. diff 원본 커밋 목록에서 확인 가능. "중복" 지적은 근거 없는 오류이므로 삭제.
