## 코드 리뷰 (diff 기반)

### 🔴 버그 / 데이터 오염 위험

**1. ProcessPoolExecutor 예외 미처리 — adni.py, aju.py**
```python
result = fut.result()  # worker 예외가 그대로 전파
```
`as_completed` 루프 안에 `try/except Exception` 없이 `fut.result()`를 호출하므로 **단 1개 세션 실패가 전체 배치를 중단**시킬 수 있다. adni.py(`run_all` for 루프)·aju.py 모두 diff에서 동일 패턴이 확인된다. nacc.py는 패치가 잘려 직접 확인 불가하나 동일 구조로 작성되어 있다고 봐야 한다. `as_completed` 루프 안에 `try: ... except Exception as e: stats["failed"] += 1; log.error(...)` 래퍼가 필요하다.

**2. AJU DTI·DWI 중복 출력 경로 — aju.py**
```python
MODALITY_MAP = { ..., "DTI": "dwi", "DWI": "dwi", ... }
```
diff에서 확인됨. 동일 세션에 `MRI/DTI/`와 `MRI/DWI/` 디렉토리가 모두 존재하면 두 태스크가 같은 `output_path`(…/dwi/…_dwi.nii.gz)를 가리켜 나중에 완료된 쪽이 먼저 완료된 결과를 덮어쓴다. 어떤 파일이 살아남을지 비결정적이며, raw_manifest resolver도 같은 경로를 조회하므로 소스 추적이 불가하다.

**3. NACC dcm2niix 입출력 경로 동일 — nacc.py**
```python
cmd = [DCM2NIIX, ..., "-o", str(tmp_path), str(tmp_path)]
```
diff에서 직접 확인됨. 압축 해제 DICOM과 출력 NIfTI가 같은 `tmp_path`에 놓인다. `force=True` 재실행 시 이전 변환 NIfTI가 같은 디렉토리에 잔존해 `glob("*.nii.gz")`에 섞이며, `max(..., key=st_size)`로 "최대 파일"을 골라도 이전 결과물이 선택될 수 있다. 출력은 별도 서브디렉토리(`tmp_path / "out"`)로 분리해야 한다.

**4. AJU MODALITY_MAP PET 항목 + 별도 PET 루프 이중 처리 위험 — aju.py**
`MODALITY_MAP`에 `"PET": "pet"`이 포함되어 있고(`EXCLUDE_DIRS`에도 없음), `iter_dicom_sessions` 내 MRI 루프는 `visit_dir/MRI/` 하위 모든 디렉토리명을 조회한다. AJU 원본 구조 기술에서 PET은 `visit_dir/PET/`(MRI 외부)에 있지만, 일부 사이트가 `visit_dir/MRI/PET/`를 가질 경우 MRI 루프와 별도 PET 루프 양쪽에서 동일 `output_path`에 변환이 시도된다. `MODALITY_MAP`에서 `"PET"` 항목을 제거하고 PET은 별도 루프에서만 처리하는 것이 명확하다.

### 🟡 수치 불안정 / 재현성

**5. "파일 크기 = 최선 볼륨" 가정 — adni.py, aju.py, nacc.py**

`session_time_years()` A4 처리 코드: `if months >= 900: return None` / 주석: `# 997 / 999 = missing-code sentinels`. 코드는 900 이상 전체를 드롭하지만 주석은 997/999만 언급한다. A4 방문 주기(≈ m0/m48/m66)를 감안하면 900–996 범위의 실제 방문은 생물학적으로 불가능하므로 실질 위험은 낮다. 그러나 코드 주석 불일치가 의도적 broadening인지 오기인지 명시 필요.
