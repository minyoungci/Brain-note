## 워크스페이스 간 연결·시너지

1. **음성 thesis의 독립 복제 — 가장 중요한 cross-WS 신호.** plant image-emb 0.662 ≈ morph 0.663 tie는 minyoung2 e01 tie와 **다른 코호트·다른 태스크(AD/CN vs prognosis)에서 동일 패턴**이다. 단일 파이프라인 artifact가 아니라 **재현된 음성**이므로, "learned가 hand-crafted를 못 넘는다"는 결론의 신뢰도를 실질적으로 끌어올린다. 이건 약점이 아니라 *발표 가능한 강한 negative result*일 수 있다.

2. **m01 positive는 thesis를 깨지 않고 강화한다(minyoung2 내부 갭의 해소).** m01에서 가치를 추가한 것은 centiloid **scalar(hand-crafted)**이고, m04에서 실패한 것은 **learned image branch**다. "m01 win ↔ m04 fail"은 모순이 아니라 *동일한 classical-beats-learned 패턴의 두 면*이다 — minyoung2 분석은 이를 "갭 분석 없음"으로 남겼으나, cross-WS 관점에선 plant tie와 정확히 같은 구조다.

3. **minyoung4 VLM 전환의 전략적 해석.** 삭제된 5종 중 korean_AD multimodal·suvr_headroom_gate는 바로 이 "deep이 tabular를 이기는가" 라인이다. **판별 가설이 프로그램 전반에서 무너지자, 비교 baseline이 없는 생성 태스크(보고서 생성)로 우회**한 것으로 읽힌다. 기회(막다른 길 탈출)인 동시에 리스크 — **음성 결과를 발표하지 않고 회피하는 것**일 수 있다. (1)의 복제된 negative result를 minyoung2+plant 공동으로 먼저 정리·발표한 뒤 VLM으로 넘어가는 편이 연구 자산을 살린다. [전략 판단은 사용자 몫]

4. **공유 부채: 누수 게이트 + TOST를 program-level 공용 유틸로.** 누수(3 WS 재발)와 등가성 검정 부재(plant+minyoung2)는 WS별로 따로 고칠 게 아니라 **fold-wise 전처리 강제 단위테스트 + bootstrap-CI/TOST 헬퍼를 한 번 만들어 세 파이프라인이 import**하면 매일 재발을 끊는다. minyoung2 fusion·plant adcn_methodology가 첫 적용 대상.

5. **데이터 소스 경계 주의.** minyoung2 m02 KDRC PET 47% 비정규화 corrupted, ADNI 다중 방문 정합 문제는 OPEN_QUESTIONS의 KDRC 라벨 모순·코호트 sex 코딩 항목과 같은 데이터 레이어다. **데이터 품질 부채가 minyoung2 멀티모달과 plant KDRC arm 양쪽에 공통 입력**이므로, 데이터 측 한 번의 정리가 두 WS에 동시 반영된다.

```python
loss = F.l1_loss(net(xm) * (1 - m), x * (1 - m))
```
표준 MAE는 마스킹된 영역(`m`)에서 재구성 손실을 계산한다. 여기선 `(1-m)`을 사용한다. `mask_vol`의 마스크 convention(`m=1`이 마스킹 복셀인지 visible 복셀인지)이 diff에 없어 확정 불가. 만약 `m=1=마스킹` 관례라면 visible 패치만 복원하는 퇴화 사전학습 → 임베딩이 무의미. **`e07_native_ssl.mask_vol` convention 확인 필수.** [외부파일 의존]

**`m04b_ssl_fusion.py` — pretrain 후 마스크 적용(transductive 누수)**
```python
t1e = pretrain_mae(T1, dev, ...)   # 전체 n=656 포함
pte = pretrain_mae(PET, dev, ...)
T1, PET, HC, CENT, y, t1e, pte = T1[m], PET[m], ...  # m = np.isfinite(y)
```
CDR 결측 피험자까지 pretrain에 포함되어 CV test fold 볼륨이 사전학습에 참여 → transductive 설정(라벨 누수는 없으나 표현 공간이 test 데이터에 맞춰짐). 논문 기술 필요.

### 데이터 누수

**`fusion_run.py` — 전역 centiloid imputation (CV loop 진입 전)**
```python
CENT = np.where(np.isfinite(CENT), CENT, np.nanmedian(CENT))
```
전체 656건 median으로 결측 대체한 뒤 CV 분할 → test fold 정보 소량 누수. fold 내 train-only median으로 impute해야 함.

**`fusion_cache.py` — ADNI 다중 방문 중복 처리 누락** *(draft에서 미검출)*
```python
lab = dict(zip(adni["subject_id"].astype(str), adni.index))
```
ADNI는 피험자당 다중 방문(longitudinal) 구조. `dict(zip(...))` 은 동일 subject_id의 마지막 행만 남김 → 이미지 경로와 라벨이 서로 다른 방문에서 왔을 가능성. 묵시적 방문 선택으로 이미지·라벨 시간 정합 미보장.

### T1 정규화 및 저장

**`fusion_run.py` — PET만 정규화, T1 없음**
```python
PET = (PET - PET.mean()) / (PET.std() + 1e-6)
# T1 정규화 없음
```
T1 강도는 스캐너·코일별로 수십 배 차이 → 학습 불안정 원인 가능.

### 수치 안정성 — 초안 주장 수정

**초안의 "bootstrap에서 pa(표준화 스케일)와 pc(원본 스케일) 혼재 시 Δ pearsonr 문제"**: **[근거부족 — 삭제]** `boot()` 함수는 `pearsonr(pa[b], y[b]) - pearsonr(pc[b], y[b])`를 계산한다. Pearson r은 선형 스케일 변환에 불변이므로 표준화·비표준화 예측값이 혼재해도 Δr 계산은 정확하다. 실질적 오류 없음.

### m01/m02 CI 일관성 — 초안 주장 부분 수정

초안은 "m02 CI가 m01처럼 seed 3개 평균이 아님"이라고 지적했으나, m01도 점추정(`msc` 3-seed 평균)과 CI 계산(`oof(..., seed=42)` 단일 seed)을 분리하여 사용한다. m02와 동일한 단일 seed OOF로 CI를 계산하므로 "m01과의 불일치"는 **과장**. 정확한 지적은: 두 파일 모두 CI에는 단일 seed 42 OOF를 사용하며, m01만 점추정에 3-seed 평균을 추가로 제공.

### ~~🔴 GRL lambda 하드코딩~~ → **[근거부족] 🟡로 강등**

초안 주장: "A.lamb 대신 1.0 사용, argparse 무력화". 그러나 실제 코드는:

```python
loss=loss+A.lamb*sitef(sh(GradRev.apply(feat,1.0)),sb)
```

`A.lamb`은 site loss 가중치로 사용됨. `GradRev.backward`에서 gradient에 `-1.0` 을 곱하므로 feat에 전달되는 net GRL 강도는 `-A.lamb × (∂sitef/∂feat)`. `--lamb` 인자를 0.5로 바꾸면 GRL 강도가 절반이 된다. **A.lamb은 유효하게 작동하며 argparse는 무력화되지 않음.** GradRev 내부 1.0과 외부 A.lamb의 위치는 관행과 다르지만 수학적으로 동치다.

잔여 우려(🟡): GradRev 내부 scale과 loss weight를 구분해 표기해 읽기 혼란을 줄 수 있음.
