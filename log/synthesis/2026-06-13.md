## 코드 리뷰 종합 (반복되는/심각한 결함)

- **🔴 leakage가 신규 코드에서 재발(class 반복).** [L1] 공변량 median imputation을 전체 집합에서 수행(OOF fold 밖). 영향은 미미하나, 어제 수술한 대형 transductive 누수(C1)와 **같은 결함 class가 새 prognosis 코드에서 다시 등장**했다. → 패턴: 새 모듈마다 leakage 재유입. leakage 점검을 사후 감사가 아닌 **commit 전 게이트**로 올려야 한다(전 WS 공통 권고).
- **🔴 통계 rigor — [B1]이 가장 심각.** 멀티시드 점추정 vs 단일시드 CI 불일치가 유의성 판정을 오염. [N1] 2포인트 polyfit + [추가-2] CN slope 분산 붕괴가 복합 → 종단 라벨 구축이 수치적으로 취약.
- **✅ 메타-긍정: critic이 초안 2건을 정확히 강등.** [B3] device 키 부재는 실질 버그 아님(저장값이 device-independent numpy) → "코드 냄새"로 격하. [N2] mask.any()=False는 확률 ≈10^−22로 "작은 배치에서 발생 가능"은 과장 → 실질 리스크 없음으로 정정. **gen→critic 분리가 false-positive를 걸러내며 작동했다** — AI-slop/finding 인플레 실패모드의 반례.
- **🟡 유지보수 리스크.** [R2] `oof`가 eval.py·rtssl_feat.py에 중복 정의(한쪽 수정 시 silent diverge). [B2] `conv_mci_ad` MCI-only가 암묵 필터에만 의존, assertion 없음. [T1] `load_pretrain` 3-tuple 보장 미확인 [불확실, diff 밖].

## 코드 리뷰 (diff 기반)

### 버그

**[B1] score와 CI 계산의 앙상블 불일치** (`experiments/prognosis/eval.py:57–67`)

`compare()`에서 `scores[k]`는 `msc_clf`/`msc_reg`의 5시드 평균 메트릭이지만, bootstrap CI에 넘기는 `oof(X[k], y, SEEDS[0])`는 시드 42 단일 예측이다. `d_vs_ref`는 멀티시드 평균 차이이고 CI는 단일시드 OOF 분포 기반이므로, 점수의 부호와 CI의 방향이 불일치할 수 있다. `p_ref`도 동일하게 `SEEDS[0]` 단일시드로 고정됨 — diff에서 직접 확인됨.

**[B2] `conv_mci_ad` 레이블에 대한 암묵적 필터링 의존** (`experiments/prognosis/data.py:44, 102`)

`conv_mci_ad`는 비MCI subject에 대해 `np.nan`으로 설정되고, `assemble()`의 `pd.notna` 필터로 제거된다. e01에서 `dx=None`으로 호출 시 MCI만 암묵적으로 남는 구조이나 명시적 assertion 없음. 의도적 설계이지만 `conv_mci_ad` task 호출 시 항상 MCI-only임을 코드에서 보장하지 않는다.

**[B3] 전역 `_FEAT_CACHE`가 device를 키에 포함하지 않음** (`experiments/prognosis/data.py:56–58`) [근거부족: 실질 버그 여부]

캐시 미스 여부는 맞다. 그러나 `_FEAT_CACHE`에 저장되는 값은 numpy array (`HC`, `COV`, RT는 npz 로드 또는 extract 후 numpy 변환)이며 device-independent하다. `get_features("cpu")`로 재호출해도 동일한 numpy 결과를 반환하므로 "silent bug" 발생 조건이 성립하지 않는다. extract_embed 구현이 diff에 없어 완전한 확인은 불가하나, 캐시 파일 경유 경로에서는 확실히 문제없음. **device 키 부재는 코드 냄새이나 실제 오작동 근거는 불충분.**

### 데이터 누수

**[L1] 공변량 median imputation이 전체 집합에서 수행됨** (`experiments/prognosis/data.py:60–62`)

`COV` 결측을 전체 subject의 `nanmedian`으로 대체 후 feature를 반환 — OOF fold 내부에서 `StandardScaler`만 train-only fit되나 imputation 단계는 test 정보를 포함한 전체 median을 사용한다. 공변량 imputation이라 실질 영향은 미미하지만 엄밀히 leakage이며, fold 내부로 이동하는 것이 원칙에 부합한다.

### 수치 불안정

**[N1] CDR-SB slope를 2포인트로 polyfit** (`experiments/prognosis/data.py:48–51`)

`ok.sum() >= 2`만 검사 — 2포인트 직선 피팅은 residual=0, 측정 노이즈에 극단적으로 민감하다. 최소 3포인트 기준 권장.

**[N2] mask.any()가 False일 때 전체 위치 loss 사용** (`experiments/rtssl_feat.py:88`) [실질 발생 불가 수준]

코드상 `F.smooth_l1_loss(pred, tgt)` 전체 재구성으로 변질되는 경로가 존재함은 사실이다. 그러나 mask shape은 `(B, NROI)`이고 NROI=95, masking 확률 0.4이므로 `mask.any()`=False 확률은 0.6^(B×95) — 배치 1이어도 ≈ 10^−22으로 사실상 발생 불가. **이론적 불일치이나 실질 리스크 없음. "작은 배치에서 발생 가능"이라는 초안 표현은 과장.**

### 재현성

**[R1] downstream 함수 내 `RNG`를 cohort 루프에서 공유** (`experiments/rtssl_feat.py:110–126`)

`np.random.default_rng(0)` 하나를 AJU→KDRC→ADNI 루프가 순차 소비한다. 각 cohort가 2000번씩 RNG를 사용하므로 cohort별 bootstrap 결과가 이전 cohort의 subject 수에 종속된다. 재실행 재현성은 있으나 cohort 간 bootstrap 독립성이 없어 개별 cohort 결과의 해석을 오염시킨다. cohort마다 `np.random.default_rng(seed)`를 독립 초기화해야 한다.

**[R2] `oof` 함수가 eval.py의 `oof_reg`와 중복 정의** (`experiments/rtssl_feat.py:112–115`)

두 구현 모두 `Ridge(alpha=10)`, `KFold(5)`, `StandardScaler` 패턴으로 동일하나 분리되어 있다. 현재 alpha=10으로 일치하지만, 한쪽만 수정 시 조용히 diverge한다.
