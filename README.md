# OBSERVATORY

뇌 MRI 치매 진단 연구의 진행 추적·지식 집약 허브. `/home/vlm`의 5개 연구 워크스페이스
(`minyoungi`, `minyoung2`, `minyoung3`, `minyoung4`, `plant`)의 활동을 매일 자동으로
집계·종합하고, 연구 이해에 필요한 지식과 인사이트를 한곳에 모은다.

이 저장소는 **2차 자료**다. 진실의 원천(source of truth)은 각 워크스페이스의 코드·리포트·
SCRATCHPAD이며, 모든 문서는 출처를 명시해 추적 가능하다.

## 구성

| 디렉토리 | 내용 | 생성 |
|---|---|---|
| `daily/` | 일일 git 활동 집계 | 자동 |
| `synthesis/` | 일일 종합 — 진행 추적·코드 리뷰·결정 필요 항목·미해결 질문 | 자동(LLM) |
| `weekly/` | 주간 회고 — 디렉토리별 실험 종합 | 자동(LLM) |
| `workspaces/<ws>/` | 워크스페이스별 카드(요약·결과·리스크·출처) + `notes/` 원본 미러 | 자동·수동 |
| `knowledge/` | 연구 개념·데이터·방법론 학습 자료 | 수동 |
| `insights/` | 5개 연구를 관통하는 횡단 인사이트 | 수동 |
| `study/` | 학습 커리큘럼 | 수동 |

## 진입점

- `DASHBOARD.md` — 한 장 요약(워크스페이스 상태·blocker·다음 게이트).
- `synthesis/`의 최신 파일 — 최근 일일 종합.
- `weekly/`의 최신 파일 — 최근 주간 회고.

## 자동화

백그라운드 스케줄러가 매일 23:50(KST)에 5개 워크스페이스의 변경을 집계하고, LLM 파이프라인
(생성 → 비평 검증 → 종합)으로 진행 상황을 정리해 커밋한다. 주 1회 주간 회고를 추가한다.
구성·복구 절차는 `tools/SCHEDULER_SETUP.md`를 참조한다.

> LLM 종합은 **검토용 초안**이다. 생성과 검증을 분리해 근거 없는 주장을 거르지만,
> 최종 판단은 원본 근거로 확인한다.

## 규약

- 문서 작성 규약: `STYLE.md` (비판적·정밀, 장식용 이모지 미사용).
- 상태 표기: `✅` 확정 · `❌` 반증 · `🟡` 잠정 · `⚠️` 주의 · `[VERIFY]` 미검증.
