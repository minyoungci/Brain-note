# OBSERVATORY

뇌 MRI 치매 진단 연구의 진행 추적·지식 집약 허브. `/home/vlm`의 5개 연구 워크스페이스
(`minyoungi`, `minyoung2`, `minyoung3`, `minyoung4`, `plant`)의 활동을 매일 자동으로
집계·종합하고, 연구 이해에 필요한 지식과 인사이트를 한곳에 모은다.

이 저장소는 **2차 자료**다. 진실의 원천(source of truth)은 각 워크스페이스의 코드·리포트·
SCRATCHPAD이며, 모든 문서는 출처를 명시해 추적 가능하다.

## 여기서 시작 (학습 경로)

처음 보거나 복습한다면 이 순서로 읽는다. 위에서 아래로 갈수록 더 구체적·최신이다.

1. `DASHBOARD.md` — 5개 워크스페이스의 역할·상태·다음 게이트를 한 화면으로.
2. `learn/curriculum.md` — 무엇을 어떤 순서로 공부할지(의존성대로 정리된 커리큘럼).
3. `learn/knowledge/` — 데이터·평가 프로토콜·방법론의 기초 지식.
4. `learn/insights/MUST_KNOW.md` — 반복 실수를 막는 횡단 핵심 교훈.
5. `workspaces/<NN_ws>/OVERVIEW.md` — 워크스페이스별 기술 블로그(코드·실험을 글로 정리).
6. `log/synthesis/`·`log/weekly/`의 최신 파일 — 지금 무슨 일이 일어나고 있는지.

## 구성

세 구역으로 나뉜다 — **learn**(직접 쓰는 공부 자료) · **log**(매일 자동 생성되는 진행 기록) · **workspaces**(워크스페이스별 정리).

| 경로 | 내용 | 생성 |
|---|---|---|
| `learn/knowledge/` | 연구 개념·데이터·방법론 학습 자료 | 수동 |
| `learn/insights/` | 5개 연구를 관통하는 횡단 인사이트 | 수동 |
| `learn/curriculum.md` | 학습 커리큘럼(읽는 순서) | 수동 |
| `log/synthesis/` | 일일 종합 — 진행 추적·코드 리뷰·결정 필요·미해결 질문 | 자동(LLM) |
| `log/weekly/` | 주간 회고 — 디렉토리별 실험 종합 | 자동(LLM) |
| `log/daily/` | 일일 git 활동 집계 | 자동 |
| `log/changelog/` | 허브 구조·도구·설정 변경 노트(블로그형) | 자동(LLM) |
| `workspaces/<NN_ws>/` | `OVERVIEW.md` 기술 블로그 + 카드(요약·결과·리스크·출처) + `notes/` 원본 미러 | 자동·수동 |
| `tools/` | 자동화 스크립트(스케줄러·노트 미러·LLM 파이프라인) | — |

진입점 단축: `DASHBOARD.md`(현황) · `log/synthesis/` 최신(최근 일일 종합) · `log/weekly/` 최신(최근 주간 회고).

## 자동화

백그라운드 스케줄러가 매일 23:50(KST)에 5개 워크스페이스의 변경을 집계하고, LLM 파이프라인
(생성 → 비평 검증 → 종합)으로 진행 상황을 정리해 `log/` 아래에 커밋한다. 주 1회 주간 회고를 추가한다.
구성·복구 절차는 `tools/SCHEDULER_SETUP.md`를 참조한다.

> LLM 종합은 **검토용 초안**이다. 생성과 검증을 분리해 근거 없는 주장을 거르지만,
> 최종 판단은 원본 근거로 확인한다.
