# STYLE — OBSERVATORY 문서 작성 규약

이 허브의 모든 문서가 따르는 편집 규약. 갱신·신규 작성 시 준수한다.

## 언어·어조

- **언어**: 한국어 본문 + 영어 기술 용어(고유명·지표명은 원어 유지: LOCO, AUROC, nuisance, transport 등).
- **어조**: 비판적·정밀. 사실과 판단을 분리한다. 구어체·감정적 수사·과장 금지.
  - 지양: "사람을 죽인", "머리에 박기", "양날의 검", "냉정하게", "때릴까".
  - 지향: "반복적으로 오류를 유발한", "우선 숙지", "장단이 공존", "현황", "지적될 지점".
- **단정 회피**: 검증 안 된 주장은 단정하지 않는다. "잘 된다" 금지 → 조건부 검증 기준으로 기술.

## 상태 표기 (전 문서 공통)

| 글리프 | 의미 |
|---|---|
| ✅ | 확정 — 산출물/검증으로 뒷받침됨 |
| ❌ | 반증(falsified) — 가설이 데이터로 기각됨 |
| 🟡 | 잠정 — 부분 근거 또는 미완 검증 |
| ⚠️ | 주의 — 함정·리스크·경계 |
| `[VERIFY]` | 미검증 — 출처 확인 또는 재현 필요 |

장식용 이모지(📦🗑🔴🟠🔵 등)는 사용하지 않는다. 위 기능적 글리프만 허용.

## 문서 헤더 형식

각 문서 상단:

```
# <제목>

> **목적:** <한 줄>  ·  **출처:** <원본 경로/근거>  ·  **갱신:** YYYY-MM-DD
```

워크스페이스 카드는 추가로 `(커밋 <7자리>)` 또는 `(SCRATCHPAD mtime)`를 갱신일에 병기.

## 구조 규약

- 섹션 제목은 명사구. 번호는 필요한 경우에만.
- 수치는 가능하면 출처와 함께(`AUROC 0.673 [.645,.699] — reports/...`).
- 표를 우선(나열형 사실은 표로). 표 헤더는 간결하게.
- 워크스페이스 카드 4종 골격 고정: `README`(요약) / `findings`(결과) / `risks`(약점) / `sources`(출처).

## 사실 무결성 (편집 시 불변)

- 수치·경로·커밋 해시·`[VERIFY]` 태그는 편집 중 **변경 금지**. 어조·구조만 정제한다.
- 새 사실을 추가하려면 출처를 명시하고, 미확인이면 `[VERIFY]`를 붙인다.
- 이 허브는 2차 자료다. 원본(각 워크스페이스 SCRATCHPAD/report/코드)이 진실의 원천.
