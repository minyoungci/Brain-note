# Consortium Common Data Matrix

Generated by `docs/context/build_consortium_common_data_matrix.py`.
This is subject-level metadata/path EDA only. It does not load image arrays, create splits, preprocess, or train.

## Bottom Line

- All four NIfTI datasets have age, sex, T1/T1c-or-post/T2/FLAIR structural core, and subject-level structural MRI coverage.
- Scanner vendor/field-strength metadata is available for 1,608/1,636 subjects and should be reported or balanced in splits.
- The most defensible first supervised target remains structural MRI IDH prediction: 1,457/1,636 subjects after conflict exclusion.
- MGMT is feasible but second-stage: 815/1,636 subjects.
- Survival/PFS, advanced imaging, and radiology-histopath pairing are not all-consortium common and require separate protocols.

## Recommended Common Schema for First Protocol

| Field group | Use | Notes |
|---|---|---|
| Structural MRI core | Input | T1, T1c/T1-post, T2, FLAIR; full header audit required before preprocessing. |
| Age, sex | Covariates/reporting | Available in all subject-level rows. |
| Scanner vendor, field strength | Confound reporting/split balance | 1,608/1,636 subjects; unknown scanner metadata must be reported. |
| IDH binary | First target candidate | 1,457 usable no-conflict subjects; strong dataset/scanner imbalance. |
| Segmentation | Optional variant | 1,617 subjects; one zero-byte segmentation path requires policy. |
| MGMT binary | Second target | 815 usable no-conflict subjects. |

## Machine-Readable Outputs

- `consortium_common_data_matrix.csv`: long feature-by-dataset availability table.
- `consortium_common_data_wide.csv`: one row per feature with dataset-specific subject counts and percentages.
- `consortium_selected_distribution.csv`: selected value distributions for clinical/scanner/label fields.
