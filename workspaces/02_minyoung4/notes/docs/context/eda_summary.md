## Common-Use Recommendation

1. Safest cross-consortium baseline: subject/session-level structural MRI (`T1/T1c/T2/FLAIR`) plus tumor segmentation where available, with age and sex.
2. Strong clinical/molecular extension: IDH and MGMT can be used across all four major imaging cohorts, but coverage differs and value vocabularies need harmonization.
3. Diagnosis/grade are available in UTSW, MU, and UCSD; UPENN is GBM-only, so it should be treated as a GBM cohort rather than a source for diagnosis/grade variation.
4. Survival/progression outcomes are not universal: OS is usable in MU, UCSD, and UPENN; UTSW lacks outcome columns in the current local metadata.
5. Advanced imaging is not universal: diffusion/perfusion is strong in UCSD and UPENN, but not a shared baseline with UTSW/MU.

## Leakage / Modeling Risks

- Split must be subject-isolated. MU and UCSD have repeated timepoints/sessions; UPENN has multiple scan IDs and both DICOM/NIfTI representations.
- Consortium/site/scanner can dominate labels. Any pooled model needs site/consortium reporting and scanner metadata checks.
- Molecular variables use different vocabularies and missing-value conventions; harmonization must be explicit before modeling.
- Outcome definitions are not aligned yet: diagnosis-based, survival-based, progression-based, and treatment-response tasks require separate cohort rules.
- This EDA did not inspect NIfTI shape/affine/orientation/spacing or DICOM pixel headers; that requires a separate read-only image-header audit.
