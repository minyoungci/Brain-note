# 연구 작업 지침

## 데이터 사용 규칙
실험에서 데이터를 불러오거나 활용할 때는 **반드시 아래 manifest 파일을 참조**한다.
디렉토리를 직접 글롭하거나 개별 스캔 경로를 하드코딩하지 말고, 항상 이 manifest를 통해 경로·라벨·메타데이터를 읽는다.

- `/home/vlm/data/preprocessed_official/official_manifest_full_n4_real_final.parquet`
- `/home/vlm/data/preprocessed_official/korean_multimodal_manifest.csv`

이 두 파일이 실험 데이터의 단일 진실원본(single source of truth)이다.

## 임상 엑셀(clinical) 파싱
임상 엑셀 파일을 가져올 때는 **매우 디테일하게 파싱하여 누락되는 데이터가 없도록** 한다.
- 파싱 실패·예외가 난 행을 조용히 버리지 말 것. 반드시 로그로 남기고 보고한다.
- 병합셀, 다중 헤더행, 결측 표기 혼재(공백·`.`·`-`·`9999`·`NA` 등)를 명시적으로 처리한다.
- 파싱 전후 행 수와 컬럼별 비결측 수를 비교해 데이터 손실 여부를 검증한다.

## 코드 품질
- 코드 최적화를 항상 고려한다(불필요한 연산·중복 I/O·메모리 낭비 제거).
- 코드 검증 단계를 철저히 거친다. 작성한 코드는 검증 없이 실험에 투입하지 않는다.

## 연구 방향성
낙관적·이상적 제안을 하지 않는다. 데이터 한계·리소스 제약·선행연구와의 차별성을 먼저 따지고,
약점과 실패 가능성을 명시한 **현실적이고 비판적인** 방향만 제시한다.

## 실험 결과 피드백
실험 결과가 나오면 단일 시각으로 끝내지 말고, **research-critic** 와 **professor** 에이전트를 호출하여
통계적 타당성·방법론적 결함·선행연구 대비 기여 관점에서 교차 검토를 받는다.

## 작업 기록 (SCRATCHPAD.md)
수행한 작업이 있을 때마다 `SCRATCHPAD.md`를 갱신하여, 새 세션을 시작해도 진행 상황을 즉시 파악할 수 있게 한다.
- **세션 시작 시 `SCRATCHPAD.md`를 먼저 읽어** 직전까지의 진행 상황을 파악한 뒤 작업을 이어간다.
- 작업(실험·코드 변경·결정 등)을 수행할 때마다 무엇을 왜 했는지, 결과와 다음 할 일을 기록한다.
- 기록은 새 세션이 맥락 없이도 이해할 수 있을 만큼 구체적으로 남긴다.
