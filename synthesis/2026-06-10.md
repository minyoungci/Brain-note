## 🔴 주목·결정 필요 (우선순위순)

2. **39.4% 커버리지 + NaN 모호성으로 manifest 저장.** NaN = "모달리티 없음" vs "DICOM 미변환"이 구분 불가. **놓쳤을 가능성**: minyoung4 G0 게이트의 "1,857 missing subj"가 진짜 결측이 아니라 **미변환 AJU·ADNI·NACC와 동일 집합일 수 있다** — 그렇다면 9.4GB ROI 추출 게이트의 작업량 산정 자체가 잘못됐다. 가장 실행가치 높은 교차 확인 [불확실, 확인 가능].
3. **subprocess 로깅 미전파(adni/aju/nacc).** Linux fork에선 상속되나 spawn/Windows에선 변환 실패 경고가 묵살 → DICOM 대량 배치 시 실패가 무성으로 누락되어 "변환 완료" 오판. 배치 실행 전 smoke-test 필수.
4. **AIBL `raw_t1_path` = skull-stripped(`native_t1w_hdbet.nii.gz`).** 다운스트림이 raw로 가정하면 이중 brain extraction. 컬럼 개명(`ref_t1_path`/`preproc_t1_path`) 또는 `is_skull_stripped` 플래그 — 결정 필요.
5. **코드리뷰 6건 자체가 미검증 가설.** 분석은 정적 read 기반이고 질문5에 "critic 실패, 미검증"이 명시돼 있다. **놓쳤을 가능성**: 종합가 입장에서 위 버그들을 "확정"으로 승격하지 말 것 — 재현 스크립트로 검증 후 등급 확정. 현재는 우선순위 높은 *후보*다.

## 놓친 것·결정 필요

**1. `oasis.py` 리뷰 불가 — diff 잘림**
diff 패치가 `preprocessing/raw_manifest/resolvers/oasis.py` 의 `new file mode` 선언 직후 잘렸다. OASIS는 FLAIR(665세션)·DWI(별도 BIDS tree)를 처리하는 가장 복잡한 resolver인데 코드를 확인할 수 없다. 커버리지 숫자(FLAIR 25.9%, DWI 13.2%)가 `oasis.py` 결과를 포함하는지 불명확하다. [불확실]

**2. NACC ses-1 274세션 해결 계획 없음**
`nacc.py` resolver, `dicom_to_nifti/README.md` 모두 "requires a separate lookup table (TODO)"로 처리됐다. NACC 전체 세션 수 대비 비율이 불명확하고[불확실], 이 274 세션의 임상 데이터 품질(완전한 labels 포함 여부)도 확인되지 않았다. 방치하면 train/eval split 시 silent exclusion이 된다.

**3. DICOM 변환 미실행 → raw_t1 커버리지 39.4% 상태로 manifest 저장**
`--verify` 기준으로 11,947 경로가 검증됐지만 AJU·ADNI·NACC의 NaN은 DICOM 변환 전임을 의미한다. 이 상태에서 manifest를 downstream 학습 코드에 넘기면 NaN = "해당 모달리티 없음"으로 해석될 수 있다. 변환 완료 전 학습코드에 raw_\*_path를 노출하지 않을 게이트(e.g., 컬럼별 완성도 assertion)가 없다.

**4. SCRATCHPAD.md 업데이트 없음**
CLAUDE.md 규칙 "실험 관련 커밋은 반드시 SCRATCHPAD.md 업데이트와 함께"가 명시돼 있다. research_topic에서 두 가지 연구 방향(shortcut-audit, foundation linear-probe)이 적부 판정됐으나 `roi_qc/SCRATCHPAD.md` 업데이트 커밋이 오늘 diff에 없다.

**5. Korean manifest(2,196 × 89) 배치 커밋(`33d47e2`)의 빌드 재현성**
한국 manifest 파일이 `preprocessed_official/`에 "배치"됐지만 이 파일을 생성한 스크립트가 오늘 diff에 없다. 언제, 어떤 입력으로 생성됐는지 추적 불가 [불확실].
